CommScript GUI - Output Area

The output area is used to show data received from the connected device as well as text that is generated from scripts and also results of evaluations entered in the input line.

When CommScript is in "AT_CMD" mode (which is a mode suited for communicating with modems through AT commands), it will split the incoming data at line feed ('\n') characters. After splitting, possibly existing carriage return ('\r') characters at the beginning or end of a line are deleted. Finally, all empty lines are deleted. At the moment, there is no way for you to change this mode. (Actually, when sending files by XModem protocol another mode is required internally, but this will be switched automatically...)

The output from the device or results of evaluation will be logged with time stamp by calling the function log_output( tag, str ). If you are interested in more details, please see the Script section of the documentation. The tag is also a string for additional information which will be appended together with a '-' directly after the time stamp. If tag is empty, no '-' will be appended after the time stamp. The resulting time stamp+tag is enclosed in brackets [...] and separated from the received output / result by a space character.

Currently, there are two tags you might come across: 'F' (buffer flush: there was no new line character received, and there was no other data received for 1 second. In this case the contents of the receive buffer will be treated as if it was a complete line, but if you see a line prefixed with [hh:mm:ss-F] you will know that this line was not terminated by a new line character.) and 'eval' (which simply means, the output was generated by an evaluation rather than from the connected device.)

You are allowed to type in the output area. What you type there is not sent to the connected device! This feature is for example very useful to insert comments - and you even may do so while there is more output appended to the end of the text, or when the connected device is off, or if the port is closed.

Special keys when cursor is in the output area:

  • <Ctrl-S> will save input, output, and setup

  • <Escape> will set focus to input line

  • <Scroll Lock> will toggle automatic scrolling; most helpful when you are typing into the output area... When automatic scrolling is off, text will be appended without jumping to the end, the setting also applies to input history, but is less critical there. Please note: the setting on/off is saved in the setup, it will be preserved even if the program is closed!

  • <F2> jump to previous regular expression match (see note below)

  • <F3> jump to next regular expression match (see note below)

The other editing key bindings are the standard Tk bindings for the Text object.

Note: Pressing <F2> / <F3> will trigger re-evaluation of the most recent search operation, so that search matches within anything that might have been added to the output area will also be highlighted and may be jumped to... (As a result, the input line will be set to the last query. Any changes to the input line will be lost if not executed before pressing F2 / F3 in the output area.)
