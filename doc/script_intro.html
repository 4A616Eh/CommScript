Introduction to CommScript Scripts

Basically, a CommScript script is a Python program. You can create very powerful scripts for practically any purpose, including operating system functions, and even opening one or more additional serial ports! For more information on the Python language, please see the Python 2.4.3 help file! Please note that quite a few modules are included with CommScript, but not necessarily all of those documented in the help file. If you are new to Python, the help file also contains a good tutorial explaining details on control flow and data structures. You will also find a complete language and library reference.

You do not need to install Python in order to use CommScript! On the other hand the interpreter could help you to test simple expressions to use in CommScript scripts... If you choose to install Python, make sure to also install PythonWin which comes with a nice user interface and is much easier to use than the command line version :-)

Very important: The script runs in a separate thread. It must exit when the script is stopped which will occur when the user clicks the 'Stop!' button or changes the main COM port settings. It may exit any time on its own, in which case the 'Run...' button will be shown again.

Let's take a quick look at how some C/C++ code fragments will look like in a CommScript script... 

  int i, j;
  unsigned char str[200];
 
  global i, j, str

In Python, variables are declared without type. The variable could even be assigned a value of one type and later a value of another type... The keyword “global” means that the variable is in the global scope of the program. If in some function the value of a global variable is changed, you will need to declare this variable as a global variable in that function!

  i = 10; j = 10;
  strcpy(str, "Hello World");
 
  i = j = 10
  str = 'Hello World'

You don't need to put a semicolon at the end of a line. Like in C you may put several statements separated by semicolons into one line of code. Assignment of the same value to more than one variable is easier... and string handling, too... Note: strings may be enclosed in single ('...') or double quotes ("...").

  if ( i && strlen(str) ) {
    ...
  } else {
    ...
  }
 
   if i and str:
     ...
   else:
     ...

No curly brackets (or other explicit syntax) is used to indicate which statements belong to then and else blocks. Just the use of indentation will group statements to blocks! It is a good idea to not mix tab and space characters to indent lines... Choose one or the other and stick with it. It might save you from potential trouble.

  for ( i=0; i<10; i++ ) {
    printf( "i = %d\r\n", i );
    ...
  }
 
   for i in range(10): 
     output( "i =", i )
     ...

The output function will print into the output area of the CommScript window. Please note that this may be delayed some milliseconds, since modifiying GUI elements is only allowed in the main thread - and the script does run in a separate thread - so there is some passing of data involved, and the main thread must regain execution... If you are creating even more threads in the script, which also use output() then the text may show up in an order you might not expect...
