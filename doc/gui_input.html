CommScript GUI - Input History and Input Line

I am using the words "input" for data entered by the user and may be sent to the connected device and "output" for the data coming from the device.

CommScript has two GUI elements dealing with input that are closely linked: the input line to enter data (e.g. AT commands to a modem) and the input history in which all inputs are logged with time stamp.

The input line is used to edit a text string until you are ready to send/execute the data/input line command. When you are done editing press <Enter>! The contents of the input line will then be appended to the input history. If the port is currently open and the text in the input line does not start with one of the following two-character prefixes, the text is written to the port, followed by '\r\n' (CRLF).

  1. =#
    Convert and send the following string from hex to binary.
    Example: =#41540D0A will send AT\r\n

  2. =$
    Send the following string after conversion from Python representation. Please note: '\r\n' (CRLF) will not be appended.
    Examples: =$AT will just send AT, =$AT\r\n will send AT followed by CRLF.

  3. =>
    Executes a Python expression.
    Examples: =>global a;a=5 will define a global variable a with value 5; =>set_title('Hello World') will call a function defined in CommScript, this will set the window title...

  4. =?
    Evaluate a Python expression.
    Examples: =?hex( ord('A') ) will return '0x41' in the output area, the time stamp will include '-eval' so you know that this was no output from the device; =?3*(5+20) will perform this simple calculation; =?a will print value of global variable a or an exception if not defined.

  5. //
    Highlight a Perl-style regular expression in the output area.
    Example: //ERROR will highlight all occurences of 'ERROR' in the text. The highlighting will be updated everytime something is typed in the input line. It will not be automatically applied to new text received! Please note: If there is plenty of text in the output area, the program may need to compute quite some time, especially if a search will match many occurences. If you would like to search for a certain string, you should probably enter the string first and then insert // at the beginning!

After <Enter> is pressed the contents of the input line remains in place. So you may send the same data once more by just pressing <Enter> again! Or you may modify the previous data and send that. The contents will be deleted and replaced by a new input, if <Enter> is followed by a regular character (no cursor key, delete, or mouse click inbetween).

Special keys when cursor is in the input line:

  • <Enter> you already know! If input line starts with // (regular expression search): Will set focus to output area and place cursor at first match.

  • <Up> and <Down> will scroll up and down the history

  • <Esc> if input line does not start with // : will clear current selection in list, so next <Up> will jump to end of list again. If input line starts with // : will set focus to output area (current cursor position is preserved).

  • <F11> will delete the current selected line from the history

  • <F12> will clear the input line

  • <Ctrl-S> will save input, output, and setup

  • <Scroll Lock> will toggle automatic scrolling; most helpful when you are typing into the output area... When automatic scrolling is off, text will be appended without jumping to the end, the setting also applies to input history, but is less critical there. Please note: the setting on/off is saved in the setup, it will be preserved even if the program is closed!

Using the mouse to select from input history:

  • Left Button, single-click will copy the contents of the clicked line into the input line

  • Left Button, double-click will copy and immediately execute input line (just if <Enter> was pressed)
