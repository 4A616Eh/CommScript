CommScript Script Library Reference

In addition to a wide selection of included Python standard libraries which may be used in scripts, there are some functions built into CommScript that are always available to scripts.

Plese note: Functions that are shown in yellow must not be called directly in the script thread. They may be called in a script indirectly from the main thread by using fun_call() (see below). Those functions could also be used directly from the input line, if the input line is started with => or =? (see GUI Input)

dialog_*(setup_names,title,filetypes)
    Please note: All dialog box functions are blocking the script thread execution until the user input is done. Also, the script may not be stopped while a dialog box is open. setup_names is a tuple e.g. ('my_path1','my_file1') and specifies under which names in the global setup (saved with Ctrl-S or on exit of the CommScript program) the selected path and filename should be stored. You may just invent any names, and when the script runs again, giving the same names again, the saved values will be used as preset values when the dialog opens. The following names are used by the main program: dwl_path, dwl_file, in_path, in_file, out_path, out_file, script_path, script_file. You may wish to use those on purpose for example the dwl* ones to get access to the saved setting of the "X" (XModem send) button of the GUI. filetypes is a list of description/filter pairs, e.g. [('all files', '*.*')].

dialog_open(setup_names,title,filetypes)
    Open filename selection dialog for open file operation. See comments above.

dialog_saveas(setup_names,title,filetypes)
    Open filename selection dialog for save file operation. Note: If the selected file exists it will ask for confirmation to overwrite. See comments above.

edit(filename[,line number])
    Open an editor window and load the specified file. A filename is required! If the file does not exist an empty editor window will be opened - and the file will be created once saved by the user. If the file is already open in an edit window, this window will be moved to the front. Optionally, a line number may be given. Then the cursor will be set to this line and the line will be highlighted

edit_close(filename)
    Close editor window. A filename is required! If the file is not open, this function will do nothing. If the file was modified and not yet saved, a message box appears asking how to proceed...

input_add(line)
    Will add a line to input history. Requires fun_call()! Note: If time stamp is not disabled, the line will be time stamped automatically. See also: log_tsf_set()

input_bind(event,callback)
    Register a callback function for an event in input history. Requires fun_call()! For example: input_bind( '<F1>', f1_event ) with a definition of f1_event as follows: def f1_event( event ): [...] will do the actions defined in f1_event() when F1 is pressed while focus is on the input history.

input_clear()
    Clear input history. Requires fun_call()!

input_load([filename])
    Load input history from file. Requires fun_call()! If filename is omitted or None or '' (empty stirng) is passed, a file selection dialog box will appear. In order to avoid compatibilty problems, the file to load should have been saved with input_save().

input_save([filename])
    Save input history to file. Requires fun_call()! If filename is omitted or None or '' (empty stirng) is passed, a file selection dialog box will appear.

inputline_bind(event,callback)
    Register a callback function for an event in input line. Requires fun_call()! For example: inputline_bind( '<F1>', f1_event ) with a definition of f1_event as follows: def f1_event( event ): [...] will do the actions defined in f1_event() when F1 is pressed while the cursor is visible in the input line.

inputline_get()
    Return text currently in input line. Requires fun_call()!

inputline_set([text])
    Set text as contents of the input line. Requires fun_call()! Please note: The input line is not executed. Passing no parameter or None will clear the input line.

log_filter_hex(str)
    Convert a string to a sequence of 2-digit hex codes. May be useful with log_filter_set().

log_filter_none(str)
    Return str. This is the default log filter - which will leave the log output at as it was passed to the log_output() function.

log_filter_set(f)
    Set log filter to some function f. This function must take a string as the only parameter and return a string.

log_output(tag,str)
    Add a string with time stamp and/or tag to the output area. The tag may be used to identify the source/reason for the log output and will be appended after the time stamp format separated by a '-'. (If time stamp is disabled, '-' before the tag is not used.) The time stamp/tag is enclosed by '[' and '] ' followed by the str which is processed through the selected log filter. The output is generated by the output() function. Please also see comments there... Note: log_output() is also used internally to log the data received from the serial port and to display results of evaluations from the input line. The tag will be added enclosing the generated output, so that the text will appear colored (if there is a definition for this tag - see: tag_color()).

log_tsf_set(str)
    Set the time stamp format for log_output() and input history. Please do not use '[' and ']' in the format. If no parameter is passed, the time stamp will be disabled. Otherwise str is used for time.strftime() as format with the local time. Default time stamp format is: '%H:%M:%S'. Please see section 6.11 time -- Time access and conversions of the Python documentation for a complete list of directives that may be used.

fun_call(f[,param[,result_callback]])
    Call a function from MainThread instead of ScriptThread. Actually, the passed parameters are added to a list of GUI updates that are periodically performed in the main thread. When the list entry is processed in the main thread, the function f will be called, if specified with the parameter param. If a result_callback function was passed to fun_call(), this function will be called with the result returned by f. Please note: You should not block the main thread (e.g. long sleep()s, blocking communication, etc.) otherwise the GUI might not work smoothly or get completely locked up. Also, any unhandled exceptions will be caught and no further action is taken!

msgbox_*(title,message)
    Please note: All message box functions are blocking the script thread execution until the user input is done. Also, the script may not be stopped while a message box is open.

msgbox_showinfo(title,message)
    Display a message box with information icon and OK button. Returns "ok" when clicked.

msgbox_showwarning(title,message)
    Display a message box with warning icon and OK button. Returns "ok" when clicked.

msgbox_showerror(title,message)
    Display a message box with error icon and OK button. Returns "ok" when clicked.

msgbox_askquestion(title,message)
    Display a message box with question mark icon and Yes / No buttons. Returns "yes" or "no" depending on which button was clicked.

msgbox_askokcancel(title,message)
    Display a message box with question mark icon and OK / Cancel buttons. Returns True or False depending on which button was clicked.

msgbox_askyesno(title,message)
    Display a message box with question mark icon and Yes / No buttons. Returns True or False depending on which button was clicked.

msgbox_askretrycancel(title,message)
    Display a message box with question mark icon and Retry / Cancel buttons. Returns True or False depending on which button was clicked.

output(*obj)
    Print the passed parameters (any number of any Python objects) to the output area. All values are generated in Python representation (using repr()) and are separated by spaces. Example: output( version, 'Hello World!', 42, range(3) ) would print something like: "CommScript 2.00 Hello World! 42 [0, 1, 2]" (without the quotes, and of course depending on the version number...). Note: This output is safe to be encoded in the file since non-(7bit)-ASCII values and special characters are encoded in the representation! If the last parameter is generated with the tag() function, the corresponding tag will be added enclosing the generated output, so that the text will appear colored (if there is a definition for this tag - see: tag_color()).

output_bind(event,callback)
    Register a callback function for an event in the output area. Requires fun_call()! For example: output_bind( '<F1>', f1_event ) with a definition of f1_event as follows: def f1_event( event ): [...] will do the actions defined in f1_event() when F1 is pressed while the cursor is visible in the output area.

output_clear()
    Clear output area. Requires fun_call()!

output_curline_get()
    Return line of text at current cursor position. Requires fun_call()! Note: a new line character at the end of the string is not included.

output_curpos_get()
    Return current cursor position in output area as a pair ( x, y ) indicating the cursor's column and row. Requires fun_call()!

output_curpos_set(x,y)
    Set cursor position in output area. Requires fun_call()! Note: First column of a row is 0, first row is 1.

output_line_get(y)
    Return one line of text in output area. Requires fun_call()! Note: a new line character at the end of the string is not included. Actually, this returns the same text as output_text_get( 0, y, 0, y+1 )[:-1] would do - but it doesn't look so ugly...

output_load([filename])
    Load contents of output area from file. Requires fun_call()! If filename is omitted or None or '' (empty stirng) is passed, a file selection dialog box will appear. In order to avoid compatibilty problems, the file to load should have been saved with output_save(). If a file filename.tags.p does exist, it will be loaded and the tags restored. Also, the color definitions will be copied to the global setup.

output_save([filename])
    Save contents of output area to file. Requires fun_call()! If filename is omitted or None or '' (empty stirng) is passed, a file selection dialog box will appear. If there are colored areas in output defined by tags, an additional file (filename.tags.p) will be written. This contains information on the colored areas and the color definitions. These will be restored on output_load(), so that the text looks the same after saving and loading.

output_str(*obj)
    Print the passed parameters (any number of any Python objects) to the output area. All values are converted to strings using str() and are separated by spaces. The example for output() above will result in the same output using output_str(). The difference is that for example output( '\n\nHello World!\n\n' ) would actually do the new lines and not printing those as \n. Note: This output is NOT safe to be encoded in the file. Saving the file may fail on some 8-bit characters if not available in the fixed, preset default encoding iso-8859-15. If the last parameter is generated with the tag() function, the corresponding tag will be added enclosing the generated output, so that the text will appear colored (if there is a definition for this tag - see: tag_color()).

output_text_get(x1,y1,x2,y2)
    Return text in output area between two positions specified by column and row. Requires fun_call()! Please note: If line breaks are included, the \n charcter is used. First column of a row is 0, first row is 1.

save()
    Save input history, output area, and setup. Requires fun_call()! This will actually work just as if <Ctrl-S> was pressed. May be used to extend CommScript with auto-save functionality. See: examples/auto_save.cs

script_autostart_set([filename])
    Run a script when CommScript is started. Passing no parameter or None will disable autostart.

script_run([filename])
    Run a script. Can not be used in ScriptThread, since only one script can run at any time. Could be used in script however, if called from MainThread using fun_call(). See examples/fun_call_quick_launch.cs! This function may be called from the main thread without a paramter. Then it will display a dialog box to select filename and add the selcted file to input history. (Actually, calling script_run() like this is done when the 'Run...' button is clicked!)

script_stop()
    Stop the script. Must not be used in script, unless executed in the main thread using fun_call()

send(text)
    Send data to port using the current port settings. Will do nothing if port is OFF. The text string may contain non-printable characters specified in the Python way, so it would be a '\x01' for sending ASCII code hex 01.

sendln(text)
    Use send to send text but terminate with CRLF ('\r\n').

sendhex(hexstr)
    Use send to send binary data specifed by a sequence of 2-digit hex codes. Example: sendhex('ABACAD') would transmit the same characters as send('\xAB\xAC\xAD').

set_title(text)
    Set the window title of the CommScript main window. Note: It will be set immediately and also saved with the setup. A customized title will not be lost when the program is closed!

tag(tagname)
    Create a special object, that will add tag information in output if passed as the last parameter in an output() or output_str() call. This function is also used internally by log_output() to add the tag passed in the first parameter.

tag_color(tagname,fg_color[,bg_color])
    Define foreground and background colors to be used to display areas of text in output for which the tag with tagname has been specified. This assignment will be stored in the global setup and in the ... .tags.p file (see output_save()).

tag_color_delete(tagname)
    Delete the color definition in global setup and remove all tags with tagname in the output text.

update_progress(percentage)
    Set the value of the progress bar. Allowed values for percentage range from 0 to 100. Fractional values are possible. Any value > 0 (e.g. 0.1) will also show the int part of the percentage as text. Note: During XModem transfers, the status line and progress bar will be updated to show the progress. It will not be harmful using update_progress during transfers, but the progress bar might be updated to a different value by CommScript...

update_status_line(text)
    Set the text displayed in the status line. Note: During XModem transfers, the status line and progress bar will be updated to show the progress. It will not be harmful using update_status_line during transfers, but the text might be overwritten by CommScript...

xmodem_start([filename])
    Start XModem transfer to transmit contents of file specified by filename. If called without a parameter, a dialog box to select a file with extension .dwl will be shown. Note: During XModem transfers, the status line and progress bar will be updated to show the progress.

xmodem_stop()
    Abort XModem transfer.
