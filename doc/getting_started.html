Getting Started with CommScript

When you run CommScript, it will look for the files 'input.txt', 'output.txt', and 'setup.p' which contain the contents of input / output areas, and the global settings. If those files do not exist, the input and/or output area will appear blank and/or the program will use default settings. Those files will be created (or overwritten with new contents) when you close CommScript. You may also press Ctrl-S anytime when the cursor is visible in the output area or in the input line. (You may wish to do that from time to time, if you have data you don't want to loose. By the way, it is OK to copy or rename the generated files after a save, if you would like to file it somewhere else...)

The available ports will be also determined at startup. CommScript will check which ports (COM1 through COM49) it is able to open and it adds those to the drop down list for port selection. Since the available ports may change while CommScript is running, it will update the list every time OFF is selected. If you feel a port is not listed but should be available since you closed it within another program, just select OFF – even if the current setting is already OFF – then it should appear when you click the com port button again!

The buttons to the right of the com port button let you choose the baud rate, byte size/parity/stop bits, and flow control (the latter I haven't tested yet). Every time one of the port parameters is changed, the port will be closed and opened again with the new settings.

Please note: The communication settings are part of the global setup! So they will be saved when the program is closed and restored, if possible when the program is started again... If the previously selected port is not available at startup, the button will automatically change to OFF.

Let's assume you have selected the port parameters appropriately and a modem is connected to the port. Now you can start sending AT commands from the input line: Make sure you can see the cursor blinking in the input line near the bottom of the window. If it is not there, just click into the input line... Type in

AT

and press <Enter>! You will see your input added with time stamp to the input history on the left, and if port parameters are correct and the modem does respond, you should see

[00:00:00] OK

or if the modem does echo your input

[00:00:00] AT
[00:00:00] OK

where [00:00:00] is the current time stamp.

Please note: The output from the device currently is split at line feed ('\n') characters. After splitting, possibly '\r' characters at the beginning or end of line are deleted. Also, since empty lines do not contain a lot of information, all empty lines will be deleted. If the device does not send line feed characters, the internal input buffer will be flushed if no data has been received for 1 second. In the output area this will be indicated by -F after the time stamp.
