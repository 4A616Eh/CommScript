CommScript Overview

The main concept of CommScript is to divide what you type and the output of a device connected to a serial port is into separate areas. Actually, using CommScript you will enter the data to send to the device into a yet separate dedicated input line. The data is not immediately sent to the port, you need to press enter first, to execute the input line. This is much easier to handle than typing into a terminal emulator if a device is constantly sending data. You are always able to read what you have typed an it does not get mixed up with the output from the device, so you don't need to remember which characters you already typed and it is easier to spot and correct typos.

After pressing Enter in the input line, the input is timestamped and copied into a history list of inputs – the left part of the vertically divided window. Everything in this history may be recalled at a later time and the history will be saved when the program is closed. So you'll be able to continue when CommScript is run again.

The outputs of the device are also timestamped (so you'll be able to figure out the sequence of events if the device does not echo the received data) and logged into the right part of the window. This area actually has editing capabilities, so that you are able to enter comments – even while more output is appended to the end of the text. The text in this area is also saved when leaving CommScript and restored on next run.

Please note, that the iso-8859-15 encoding used to save the files. It will cause you trouble if you type characters that cannot be encoded in this character set! It even may happen that you cannot exit CommScript and that you do not get an error message why... I guess some improvement could be done here... (Actually, if the device does send characters > ASCII 127 they will be converted to a representation which is 7-bit clean, so you don't need to worry that garbage may come from the device and screw things up... Just be sure what you type is OK...)
